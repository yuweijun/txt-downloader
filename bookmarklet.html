🚀 How to Use

1️⃣Navigate to Target Page

Go to the webpage you want to scrape (e.g., a blog article, novel chapter, documentation page)

2️⃣Click the Bookmarklet

Click the "Scraper" bookmark in your bookmarks bar. A scraper panel will appear on the page.

3️⃣Configure Selectors

The bookmarklet will show a form with smart defaults:

  • Content Selector: Where to extract text from
  • Title Selector: How to find the page title
  • Next Link Selector: How to find the "next page" button (optional)
  • Max Pages: How many pages to follow

4️⃣Start Scraping

Click "🚀 Start" and watch it work! The scraper will:

  • Extract content from the current page
  • Find and click the "next" link
  • Repeat until max pages or no more pages
  • Download everything as a single .txt file

💡 Common Selectors

📰 Blog Articles / News Sites:

  • Content: article, .post-content, .entry-content
  • Title: h1, .post-title, .entry-title
  • Next: a.next, a[rel="next"]

📖 Novel/Fiction Sites (Chinese):

  • Content: #content, .content, #article
  • Title: h1, .title
  • Next: a:contains(下一章), a:contains(下一页)

📚 Documentation Sites:

  • Content: main, .markdown-body, .doc-content
  • Title: h1
  • Next: a.next-page, a[aria-label="Next"]
💡 Pro Tips:
  • Inspect Element: Right-click → Inspect to find CSS selectors
  • Multiple Selectors: Use commas: article, .content, main
  • Test First: Try with max pages = 1 to test selectors
  • Single Page: Leave "Next Link" empty to scrape just current page

🔧 Troubleshooting

Issue: Nothing happens when I click the bookmarklet
  • Make sure you clicked the bookmark, not just viewed it
  • Try opening the browser console (F12) to see any errors
  • Some sites block JavaScript - try a different browser
Issue: No content extracted
  • Right-click on the content → Inspect to find the correct selector
  • Try broader selectors like main or article
  • Use browser console: document.querySelectorAll('YOUR_SELECTOR')
Issue: Next page not working
  • Inspect the "next" button to find its selector
  • For Chinese sites, try: a:contains(下一页)
  • For English sites, try: a.next or a[rel="next"]

🔬 Advanced Tips

Finding the Right Selectors:

  1. Right-click on the content you want → "Inspect Element"
  2. Look at the HTML element's class or id
  3. Use #id for IDs (e.g., #content)
  4. Use .class for classes (e.g., .article-body)
  5. Use element names directly (e.g., article, main)

Testing Selectors in Console:

  1. Press F12 to open Developer Console
  2. Type: document.querySelector('YOUR_SELECTOR')
  3. If it returns an element, your selector is correct!
  4. Use querySelectorAll to see all matches
